# Research

Deep-dive research on specific topics.

## What belongs here

- Thorough investigations into a subject
- Literature reviews, comparisons, and analyses
- Collected findings with sources and references

## Organization

- **One folder per topic.** Never store research as a single loose markdown file
  at the top level of `research/`. Each topic gets its own directory.
- Inside each topic folder, store one or more markdown files covering different
  angles (e.g., `overview.md`, `sources.md`, `comparison.md`).
- Supporting non-Markdown files (PDFs, images, spreadsheets, etc.) may also be
  stored in a topic folder when they add value to the research.
- Cite sources with links where possible.

## Naming conventions

- Folder names: lowercase, hyphens as separators (e.g., `rust-error-handling/`, `llm-fine-tuning/`).
- File names inside folders: lowercase, hyphens, descriptive.

## Examples

```
research/
  rust-error-handling/
    overview.md
    crate-comparison.md
  llm-fine-tuning/
    overview.md
    dataset-preparation.md
    sources.md
```
